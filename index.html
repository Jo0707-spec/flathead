Ideen für Umsetzung

  • Direkt in Node-RED: GPS-Daten vom Raspberry per MQTT/HTTP einsammeln und im Dashboard live anzeigen.
  • LoRa fallback: Nur Kernwerte (Lat/Lng/Battery/Alarm) per LoRa übertragen, Node-RED dekodiert das Paket.
  • Hybrid: LoRa für Reichweite + WLAN/LTE für detailreiches Tracking, je nach Verbindung automatisch umschalten.
  • Geo-Fencing: In Node-RED Trigger setzen (z. B. "außerhalb Zone") und Alarm an Web/Telegram senden.

Tipp: LoRa reicht für Positions-Updates in Intervallen (z. B. 15–60s), Video sollte separat über WLAN/LTE laufen.

Live Kamera Feed

Streaming-Ansatz über Node-RED + Raspberry Pi Kamera

Livestream

Noch kein Stream konfiguriert.

Setze in app.js die URL cameraFeedUrl auf deinen MJPEG/HLS Stream.

Empfohlene Einbindung

  • Einfach: Raspberry Kamera als MJPEG Stream (z. B. über libcamera-vid + Gateway) und im <img> anzeigen.
  • Node-RED: Stream-URL als HTTP-In/Proxy in Node-RED bereitstellen, damit alles über einen Server läuft.
  • Sicher: Authentifizierung (Token/Basic Auth) und optional HTTPS, wenn extern erreichbar.
  • Performance: Für stabile Live-Ansicht besser HLS/WebRTC statt einzelner JPEG-Snapshots.
